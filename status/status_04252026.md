# Hot Dice — Project Status (2026-04-25)

## Project Overview

**Hot Dice** is a Godot 4.5 3D dice-rolling game built with Forward Plus rendering. All 3D assets are procedurally generated — no external models or textures. The project is not yet a git repository.

## Scene Breakdown

### main_menu.tscn
- Control node with CenterContainer layout
- Title label ("Hot Dice") and a "Play" button
- Attached: `main_menu.gd`

### game.tscn
- Root: Node3D (game.gd)
- DirectionalLight3D with shadows
- Bird's-eye camera (~45° angle)
- Table: StaticBody3D (green cylindrical felt, brown torus outline, circular collision walls via table.gd)
- Dice: Node3D containing 6 Die instances (Die1–Die6), each starting frozen at (0, -1, 0)
- UI: CanvasLayer with "Roll Dice" button (bottom center)

### die.tscn
- Root: RigidBody3D, starts frozen
- Size: 0.4 × 0.4 × 0.4 units, light beige color
- BoxMesh + BoxShape3D collision
- Attached: `die.gd`

### test.tscn
- Simple rectangular table (3 × 2 × 0.1 units) with a camera
- Used for isolated physics testing

## Script Breakdown

### main_menu.gd
- `_on_play_button_pressed()`: calls `get_tree().change_scene_to_file()` to load game.tscn

### game.gd
- Defines 6 DROP_POSITIONS in a 2×3 grid (X: ±0.9, Z: ±0.35, Y: 1.5) with ±0.08 random offset
- Caches all 6 dice on `_ready()`
- `_on_roll_button_pressed()`: calls `die.roll(position)` on each die

### die.gd
- Signal: `roll_completed(value: int)`
- Constants: FACE_NORMALS (6 face normal vectors), FACE_VALUES ([1, 6, 3, 4, 5, 2])
- `roll(drop_position)`: unfreezes die, sets position, randomizes rotation, applies random torque impulse (5.0 units)
- `get_face_up()`: dot-product against world UP to determine which face is up; returns mapped die value
- `_on_sleeping_state_changed()`: when die stops moving, emits `roll_completed` with face value
- `_rolling` flag distinguishes initial drop from final settle

### table.gd
- Procedurally generates 16 BoxShape3D collision segments arranged in a circle (WALL_RADIUS: 1.5, WALL_HEIGHT: 0.25, 16 SEGMENTS) to form an invisible circular bowl

## Current Game Flow

1. Launch → main_menu.tscn
2. "Play" → game.tscn
3. "Roll Dice" → all 6 dice drop with physics (gravity, torque, collision)
4. Each die settles and emits `roll_completed(value)` — **signal is not consumed by any UI**

## What Is Working

- Physics-based dice rolling with randomized torque
- Face detection via dot-product (accurate for a cube)
- Circular table with procedural wall collision
- Scene transitions (menu → game)
