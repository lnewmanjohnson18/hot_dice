## Project Overview

Hot Dice is a 3D physics-based dice roller built in Godot 4.5. All 3D assets are procedurally generated — no external models, textures, or audio. The game consists of six RigidBody3D dice that drop onto a circular felt table with procedurally generated wall collision when the player clicks Roll Dice. The project entry point is scenes/main_menu.tscn, which transitions to scenes/game.tscn on play.

Scene tree for game.tscn: Game (Node3D, game.gd) → DirectionalLight3D, Camera3D (bird's-eye ~45 degrees), Table (StaticBody3D, table.gd) → FeltMesh, FeltCollision, WallMesh, Dice (Node3D) → Die1–Die6 (instances of scenes/die.tscn), UI (CanvasLayer) → RollButton (Button), ScorePanel (VBoxContainer) → RollScoreLabel (Label), TotalScoreLabel (Label).

Scripts: scripts/game.gd (roll controller, scoring logic), scripts/die.gd (physics, face detection, roll_completed signal), scripts/table.gd (procedural circular wall collision), scripts/main_menu.gd (scene navigation only).

## Changes Made This Session

### 1. WallMesh Fix (game.tscn)

Problem: The TorusMesh used as the decorative wall rim had inner_radius=0.1, outer_radius=1.5. In Godot's TorusMesh, the tube cross-section radius is (outer-inner)/2 = 0.7 and the ring center is at (outer+inner)/2 = 0.8 from origin. This made it a thick solid disk covering the entire play area instead of a thin rim.

Fix: Changed to inner_radius=1.3, outer_radius=1.6. New ring center is at radius 1.45 (right at the felt edge, which has radius 1.4), tube radius 0.15 — a thin decorative rim with the center fully open. The actual physics collision comes from table.gd (16 BoxShape3D segments in a circle), not the torus mesh, so physics was unaffected.

### 2. Farkle Scoring Implementation (game.gd + game.tscn)

game.gd was completely rewritten. New state variables: _dice_done (int), _roll_values (Array[int]), _total_score (int). New @onready refs: _roll_button (Button at UI/RollButton), _roll_label (Label at UI/ScorePanel/RollScoreLabel), _total_label (Label at UI/ScorePanel/TotalScoreLabel).

In _ready(), each die's roll_completed signal is connected to _on_die_roll_completed. On roll button press: counter and values array are reset, button is disabled, "Rolling..." shown, then all 6 dice are triggered. _on_die_roll_completed appends the value and increments the counter; when all 6 are in, _finish_roll() is called. _finish_roll() calls _score_roll(), adds to total, updates both labels, re-enables the button.

Scoring rules in _score_roll(values: Array[int]) -> int: Check for straight (6 dice, all 6 unique values) = 1500. Check for three pairs (3 entries with count==2) = 1500. Then for each face value: if count >= 3, score base * (count-2) where base=1000 for 1s, face*100 otherwise — this gives 3-of-a-kind=base, 4-of-a-kind=2x, 5-of-a-kind=3x, 6-of-a-kind=4x. Remaining 1s score 100 each, remaining 5s score 50 each. Returns 0 (FARKLE) if no scoring dice.

game.tscn additions: ScorePanel VBoxContainer anchored to top-center (offset_left=-150, offset_top=12, offset_right=150, offset_bottom=72, anchors_preset=5). Two child Labels: RollScoreLabel (text="Roll: -") and TotalScoreLabel (text="Total: 0"), both layout_mode=2, horizontal_alignment=1, font_size=22.

## Recommended Next Steps

The most natural next feature is die selection — allowing the player to click individual dice to mark them as kept, then reroll only the unkept dice. This requires: a "kept" state on each die (die.gd), visual feedback for kept dice (color change or highlight), modifying game.gd's roll logic to skip kept dice, and a UI affordance to confirm the selection and reroll. After that: turn structure with scoring thresholds, FARKLE detection ending the turn, and a proper win condition.
